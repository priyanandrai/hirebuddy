# Contributing to HireBuddy

Thank you for your interest in contributing to HireBuddy! We welcome contributions from everyone.

## How to Contribute

### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/hirebuddy.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Commit your changes: `git commit -m "Add your descriptive commit message"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

### Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions small and focused
- Comment complex logic

### What to Contribute

We welcome contributions in these areas:

#### For Beginners
- Improve documentation
- Add more examples
- Fix typos
- Add comments to code
- Write tests

#### For Intermediate Contributors
- Add new features (see Issues)
- Improve error handling
- Optimize existing code
- Add validation logic

#### For Advanced Contributors
- Implement database integration
- Add REST API endpoints
- Build frontend interface
- Add authentication/authorization
- Implement real-time notifications

### Adding New Features

When adding a new feature:

1. Check if there's an existing issue for it
2. If not, open an issue to discuss the feature
3. Get feedback before starting implementation
4. Follow the existing code structure
5. Add documentation for the new feature
6. Update the README if necessary

### Testing

Before submitting a PR:

1. Test your code manually
2. Ensure existing functionality still works
3. Add test cases if applicable

### Pull Request Guidelines

- Use a clear and descriptive title
- Describe what changes you made and why
- Reference any related issues
- Keep PRs focused on a single feature/fix
- Be responsive to feedback

### Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Give constructive feedback
- Focus on what is best for the community

### Need Help?

- Open an issue for questions
- Check existing issues and documentation
- Reach out to maintainers
